# FixItFriday

**Weekly sessions focused on finding, understanding, and fixing security vulnerabilities in first-party code.**

Every Friday, we pick a real-world vulnerability class, spin up a purposely vulnerable application, and walk through identification, exploitation, and remediation — all in code you or your team might write.

---

## Session Flow

| Step | What Happens |
|------|-------------|
| **Current State** | Review vulnerability statistics across the org — what's trending, what's critical |
| **Demo App** | Spin up a purposely vulnerable application targeting a specific vulnerability class |
| **Code Walkthrough** | Walk through the vulnerable code and explain why it's dangerous |
| **Exploitation** | Exploit the application live with a working proof of concept |
| **Remediation** | Modify the code to the secure version and explain the fix |
| **Verification** | Re-run the exploit to confirm the vulnerability is resolved |
